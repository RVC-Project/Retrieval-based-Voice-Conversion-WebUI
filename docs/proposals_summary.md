## 提案1: 日本語SSLモデル導入

### 概要
ContentVec（英語LibriSpeech 960h）を日本語SSLモデルに置換。最有力候補は `imprt/kushinada-hubert-base`（産総研、日本語TV放送62,215h、2025年3月公開）。768次元・12層の同一アーキテクチャのためドロップイン置換可能。

### 候補モデル比較

| モデル | 出力次元 | 日本語データ | RVC互換 | ライセンス | 推奨度 |
|--------|---------|------------|---------|-----------|--------|
| **imprt/kushinada-hubert-base** | 768 | **62,215h** | **完全互換** | Apache 2.0 | **1位** |
| reazon-research/japanese-wav2vec2-base | 768 | 35,000h+ | 完全互換 | Apache 2.0 | 2位 |
| rinna/japanese-hubert-base | 768 | 19,000h | 完全互換 | Apache 2.0 | 3位 |
| Spin V2 | 768 | ContentVec微調整 | 完全互換 | - | 4位(発音精度最高) |
| mHuBERT-147 | 768 | 90,000h/147言語 | 互換 | Open | 5位 |
| WavLM Large (layer 6) | 1024 | 94,000h多言語 | 要次元調整 | MIT | 6位(話者情報最少) |
| MERT | 768/1024 | 音楽特化 | 中 | CC BY-NC-SA | 7位(歌声特化) |

### メリット
- 日本語音素表現の飛躍的向上（特に「ウ」/ɯ/、促音、ら行）
- 768次元一致のためモデルアーキテクチャ変更不要
- nadare氏により実証済み（日本語発音がクリアに）
- 少量データでの学習効率向上
- Apache 2.0で商用利用可能
- kushinadaはrinnaの3.3倍のデータ(62,215h)で学習。TV放送由来のため話者・ジャンル多様性が最高

### デメリット・リスク
- 既存RVCモデル(.pth)・FAISSインデックスとの非互換（再学習必要）
- 事前学習済みモデル(pretrained_v2)の再作成が必須
- `transformers`ライブラリの依存追加
- 歌唱ドメインとの乖離（話し言葉で学習）
- jp-HuBERT使用時はindex_rate 0.8-1.0への引き上げ必須
- kushinadaはTV放送の話し言葉で学習されているため、歌声での品質は別途検証が必要（歌唱特有の発声・ビブラート等への適応度は未確認）

### 必要ハードウェア
- **推論**: 現行と同等（VRAM追加なし）
- **学習（個別モデル）**: VRAM 8GB+、約1時間
- **事前学習再作成**: VRAM 24GB+ x 1（A100推奨）、数百GPU時間
- **短縮策**: nadare氏のBOOTH公開チェックポイント利用で事前学習省略可

### 変更対象ファイル
`infer/lib/jit/get_hubert.py`, `infer/modules/vc/pipeline.py`, `infer/modules/train/extract_feature_print.py`, `configs/v2/*.json`, `infer/lib/infer_pack/models.py`

### 概要

## 提案3: 損失関数・学習手法改善

### 概要
現行4損失を8損失に拡張し、学習プロセスを最適化。

### 改善の段階的導入

| Phase | 改善 | 効果 | リスク | 状態 |
|-------|------|------|--------|------|
| **1（即時）** | Dropout(0.1) + Weight Decay(G=0.01, D=0) | 過学習防止 | 低 | ✅ 実装済み |
| **1（即時）** | Multi-Resolution STFT損失 (c_mrstft=5.0) | 子音明瞭度向上 | 低 | ✅ 実装済み |
| **1（即時）** | bf16混合精度対応 | 学習安定化 | 低 | ✅ 実装済み |
| **2** | KLサイクリカルアニーリング | 潜在空間最適化 | 低 | 未着手 |
| **2** | EMA(alpha=0.999) | 推論品質安定化 | 低 | 未着手 |
| **3** | CosineAnnealingWarmRestarts | 学習安定化 | 中 | 未着手 |
| **3** | 勾配蓄積(4ステップ) | 実効バッチ16 | 中 | 未着手 |
| **4** | F0整合性損失 | ピッチ精度強制 | 中 | 未着手 |
| **4** | フォルマント保存損失 | 母音明瞭度 | 高 | 未着手 |

### メリット
- Dropoutだけで少量データの過学習を直接軽減（p_dropout=0は致命的）
- MRSTFT損失で子音のアタック再現性向上（Parallel WaveGANでMOS 1.36→4.06の報告）
- EMAでチェックポイント間の品質ばらつき解消
- 外部依存は`auraloss`のみ

### デメリット・リスク
- ハイパーパラメータが約15個追加（チューニングコスト）
- 学習時間20-40%増加
- 全損失を一度に導入すると切り分け困難

### 必要ハードウェア
- **Phase 1-2**: 現行 + VRAM 1GB程度（RTX 3060 12GBで可）
- **Phase 3-4**: VRAM 12GB+推奨

## 提案5: データ戦略

> 詳細: [docs/proposal_data_strategy.md](proposal_data_strategy.md)

### 概要
10分の歌声データを70-100分相当に拡張し、段階的転移学習で品質最大化。

### データ拡張パイプライン

| 手法 | データ倍率 | パラメータ | 品質リスク |
|------|----------|-----------|----------|
| ピッチシフト(±2,±4半音) | **4倍** | librosa.effects.pitch_shift | 8半音超でアーティファクト |
| タイムストレッチ(0.9-1.1x) | **2倍** | librosa.effects.time_stretch | 15%超で不自然 |
| SpecAugment | 1倍(正則化) | freq_mask=20, time_mask=80 | 適用率50%推奨 |
| TTS逆翻訳(GPT-SoVITS) | **3-5倍** | UTMOS>3.5フィルタ | TTS品質に依存 |
| **合計** | **7-10倍** | 10分→70-100分相当 | |

### 歌唱向け前処理パラメータ ✅ 実装済み

| パラメータ | 旧値 | 現在値 | 状態 |
|-----------|------|--------|------|
| Slicer threshold | -42 dB | **-38 dB** | ✅ 実装済み |
| per（セグメント長） | 3.7 s | **5.0 s** | ✅ 実装済み |
| overlap | 0.3 s | **0.5 s** | ✅ 実装済み |
| min_length | 1500 ms | **2000 ms** | ✅ 実装済み |
| ハイパスフィルタ Wn | 48 Hz | **40 Hz** | ✅ 実装済み |

### 転移学習フロー
```
ReazonSpeech(19,000h) → rinna HuBERT
  ↓
JVS-MuSiC(100話者) → マルチスピーカー事前学習
  ↓
歌声DB(きりたん+GTSinger等) → 歌声適応
  ↓

```
